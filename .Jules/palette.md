## 2025-05-14 - [Form and Button Accessibility Standards]
**Learning:** In a complex UI like the Game Detail view, many interactive elements (icon-only buttons, range inputs, and standard form controls) were missing basic accessibility hooks. Native HTML5 elements like `<input type="range">` and `<select>` require explicit `id` and `htmlFor` associations to be correctly identified by screen readers. Additionally, emojis used as button labels must be hidden with `aria-hidden` when an `aria-label` is provided to avoid redundant or confusing announcements.
**Action:** Always verify that every form control has a unique ID and a corresponding `<label htmlFor="...">`. For icon-only buttons, ensure an `aria-label` is present and wrap decorative emojis in `<span aria-hidden="true">`.
