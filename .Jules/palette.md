## 2025-05-14 - [Game Detail Header Accessibility & UX Polish]
**Learning:** In a specialized desktop manager like Pascal, many interaction points (like folder paths or branding overlays) are implemented as static elements with click handlers. These must be converted to semantic interactive elements (like `<button type="button">`) or given appropriate ARIA roles and focus indicators to be keyboard-accessible. Additionally, native HTML5 range inputs cannot represent a `null` state, so a separate "Clear" action is required for features like personal ratings to allow users to unset their data.
**Action:** Always verify that every clickable element is either a semantic `<button>` or `<a>` tag with proper focus states. For range inputs that represent optional numeric data, provide a companion "Clear" button and ensure all icon-only toggles have `aria-label`, `title`, and `aria-pressed` attributes.
